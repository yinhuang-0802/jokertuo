# A3 出牌页竞品分析与优化方案

## 1. 背景与目标
本分析基于你提供的《微乐 A3》出牌页截图，对其信息架构、交互路径、视觉风格进行拆解，并给出当前 A3 产品（本项目）的优化方向。

目标：在不改变 A3 核心规则的前提下，借鉴“欢乐斗地主式”的高可读、高反馈、低误触体验，使玩家在短时间内完成局势理解与出牌决策。

## 2. 竞品页面拆解（微乐 A3）

### 2.1 信息分区（空间组织）
- 顶部：回合结果/阶段状态（如“不出”）在视觉中心，且不与牌面冲突。
- 中部：桌面舞台（落牌、上一手展示、少量状态信息），几乎不放按钮。
- 两侧贴边：玩家信息（头像/等级/余额/剩余牌数）贴近边缘，避免遮挡牌面。
- 底部：自己的信息条 + 操作工具 + 牌型入口，形成固定操作区。

### 2.2 信息层级（优先级）
从高到低：
1) 当前回合状态（不出/轮到谁/倒计时）
2) 台面上一手/本手牌型（用于判断是否能压）
3) 玩家剩余牌数与关键提示
4) 工具与活动（记牌器、活动倍数、表情等）

### 2.3 交互路径（减少思考与试错）
- 牌型入口前置：底部“顺子/同花/葫芦/金刚/同花顺”等作为可行动作入口，而不是仅提供一个“提示”按钮。
- 工具弱侵入：锁牌/记牌器以小入口常驻；聊天/表情贴边，不遮挡台面。

### 2.4 视觉与动效（对局反馈）
- 桌面“舞台感”：桌布质感与大面积留白，使牌面成为绝对主角。
- 牌面标准化：角标/花色红黑对比强，字号层级清晰。
- 结果强反馈：不出/压住等结果以顶部大字短驻留，低成本让玩家“看见发生了什么”。

## 3. 当前产品问题归因（对照竞品）
### 3.1 遮挡与层级
- 桌面缺少强约束分区，多个信息块在同一舞台内叠加，容易互相覆盖。

### 3.2 牌面可读性
- 花色对比不足与角标信息不完整，会导致“只看见数字”的主观体验。

### 3.3 操作区干扰
- 悬浮按钮在小屏/中屏容易压住手牌，增加误触与选牌成本。

### 3.4 行动可见性不足
- “提示”没有被拆解成明确的可执行入口，玩家需要反复尝试选牌，学习成本高。

## 4. 优化方案（按优先级落地）

### P0（遮挡归零 + 牌面清晰 + 不误触）
- 固定三段式骨架：顶部状态条 / 中部台面舞台 / 底部操作区。
- 按钮永远不覆盖手牌；手牌保持单行并可横向滚动。
- 牌面标准化：角标显示“点数+花色”，红黑高对比，选中态金色高亮。

### P1（可行动作前置：像斗地主一样“点一下就能出”）
- 增加“可出组合”横滑条：单张/对子/顺子/同花/葫芦/四条/同花顺。
- 点击某牌型即可自动选中一组可走牌；再次点击可循环切换备选组合；不可用状态显示原因。
- 增加“重选/锁牌”来降低误操作。

### P2（A3 差异化：阵营信息的稳定呈现）
- 在玩家信息位稳定展示 A♠/3♠ 的暴露状态：未暴露为“?”，暴露后高亮。
- 打出 A♠ 或 3♠ 时触发轻量提示（顶部条+座位标识），让身份揭示成为关键事件。

## 5. 验收与指标
- 视觉验收：台面无遮挡（手牌可见 100%，座位卡与落牌互不压住）。
- 决策效率：轮到玩家后至首次有效选牌时间下降。
- 试错率：非法出牌拦截次数/局下降。
- 提示使用：可出组合条点击率、带来的实际出牌占比提升。

## 6. 组件映射（建议实现结构）
- `TopStatusBar`：顶部状态条（回合状态/行动方/倒计时/设置入口）。
- `TableStage`：桌面舞台容器（座位区+落牌区+动效层）。
- `Seat`：玩家信息块（头像/剩余/暴露标记）。
- `SeatPilesLayer`：按座位落点展示上一手牌。
- `HandArea`：单行叠牌/横向滚动/选牌。
- `MoveChipsBar`：可出组合横滑条（点击即选）。
- `ActionDock`：操作区（出牌/不要/提示/重选/锁牌）。
